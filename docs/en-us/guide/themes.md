# Themes

The **Themes** tab is where you manage the visual identity of your website. Like the plugins area, this dashboard lets you activate, delete, update (or auto-update), and switch themes — without needing to log into the WordPress admin backend.

## Activate or update a theme

1. Open your website in the OS Cloud panel and select **Themes**.
2. Check the current status and available version.
3. Create a [backup](/en-us/guide/backups) before a major update or theme change.
4. Complete the action, then check the homepage, navigation, forms, shop, and mobile layout.

::: warning
Changing themes can affect widgets, menu locations, templates, and styling. Test a theme change on staging where possible, and avoid scheduling it during a busy sales period.
:::

## Remove a theme

Only remove inactive themes you no longer need. Keep one current default theme available alongside the active theme for troubleshooting. Never remove an active theme or the parent of an active child theme.

If the site stops working correctly after a change, restore the backup or reactivate the previous theme and collect error details before [contacting support](/en-us/guide/contact-support).
