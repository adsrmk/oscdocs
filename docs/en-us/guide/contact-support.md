# Contact support

A complete problem description helps our support team identify the cause and appropriate solution more quickly. Use this checklist before contacting us.

## Check this first

1. Check the [OS Cloud status page](https://status.oscloud.nl/) for current incidents or maintenance.
2. Search OS Support for the exact error message or affected component.
3. Reproduce the issue once in a private browser window or on another network, unless doing so could change or delete data.
4. Record what you have already tried and the result of each step.

::: danger Never share secret information
Do not send passwords, two-factor authentication codes, recovery codes, private keys, complete database exports or API secrets. Remove personal data and secret values from screenshots and logs.
:::

## Information to include

Where possible, provide:

- your account or company name;
- the affected domain, email address or website name;
- the exact error message without paraphrasing it;
- the date, time and time zone of the issue;
- which users or visitors are affected;
- the steps required to reproduce the problem;
- recent changes to DNS, plugins, themes, code or settings;
- relevant screenshots or log entries with sensitive data removed.

## Message template

```text
Subject: [component] short description of the problem

Account/company:
Affected domain or service:
Started at (date, time and time zone):
Exact error message:
Impact on users or business operations:
Steps to reproduce:
Already tried:
Recent changes:
```

## Describe the urgency clearly

Explain the business impact objectively. For example:

- **Critical:** a production environment or essential email service is unavailable to every user.
- **High:** an important function is unavailable and there is no practical workaround.
- **Normal:** a component is partially affected or you have a configuration question without an active outage.

Marking a request as “urgent” without explaining the impact or providing error details does not help diagnosis and may lead to additional questions.
