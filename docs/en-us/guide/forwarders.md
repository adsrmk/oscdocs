# Forward email

An email forwarder automatically sends new incoming messages to another email address. This is useful for functional addresses such as `info@` or when one team manages several addresses centrally.

## Configure forwarding

1. Open [Emails](https://oscloud.nl/emails) and select the mailbox you want to manage.
2. Go to **Forwarders**.
3. Add the complete destination address and check it carefully for typing errors.
4. Save the change.
5. Send a test message from an external address to the source address and confirm delivery at the destination.

## Important considerations

- Do not create a loop in which two addresses forward to one another.
- Check whether messages must also remain in the original mailbox; this behaviour may depend on the selected setting.
- For several team members, prefer personal mailboxes or a shared mailbox over a long chain of forwarding rules.
- Forwarding to an external provider can affect spam checks and authentication. Check the recipient's spam folder as well.
- Remove obsolete forwarders immediately when an employee or supplier should no longer receive messages.

If forwarding fails, first confirm that direct delivery to the source mailbox and direct email to the destination both work. When contacting support, provide both addresses, the time of the test and any delivery error; do not share message content containing personal data.
