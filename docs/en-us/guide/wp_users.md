# WordPress Users

The **Users** section manages administrative and team access for your WordPress site.

<div class="tip custom-block" style="padding-top: 8px">
WordPress users are different from <b>collaborative users</b> in OS Cloud. WordPress users only have access to the WordPress back-end — they <b>cannot</b> access the OS Cloud panel.
</div>

<br>

## Manage accounts

- **Add new users** — Click **Add User** to grant access to a new team member. You'll need a username, email address, and password.
- **Edit details** — To update an existing user (e.g. username or email), click the **kebab menu** (three vertical dots) on the right of the user row and select **Edit**.
- **Delete access** — Permanently remove a user to revoke their access to the WordPress back-end.

<div class="warning custom-block" style="padding-top: 8px">
Deleting a user is <b>irreversible</b>. Always create a backup first — especially if the user owns published content. Removing the primary admin account can also break parts of the site if associated content is reassigned incorrectly.
</div>

<br>

## Single Sign-On (SSO)

**SSO** gives you instant access to the WordPress admin area straight from the OS Cloud panel — no manual login required. If your site has multiple administrative accounts, you can choose which one SSO logs in as.
